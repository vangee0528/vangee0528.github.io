Flutter 性能优化实践

Flutter 2026-01-28 · 12 分钟阅读

从渲染原理出发，掌握 Flutter 应用的性能调优技巧。

Flutter 渲染原理

Flutter 采用声明式 UI 和即时模式渲染。当状态改变时，Flutter 会重建 Widget 树，然后通过 RenderObject 树进行布局和绘制，最后合成到 GPU。

避免不必要的重建

使用 const 构造函数

// 好：const 避免重建
const Text('Hello')

// 差：每次重建
Text('Hello')

拆分 Widget

将大 Widget 拆分为小 Widget，只有变化的部分会重建。

class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        StaticPart(),  // 不会重建
        DynamicPart(), // 会重建
      ],
    );
  }
}

使用 const 优化 ListView

ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) {
    return const ListTile(title: Text('Item'));
  },
)

避免卡顿

使用 ListView.builder

对于长列表，使用 ListView.builder 实现懒加载。

避免在 build 中做耗时操作

// 差：每次 build 都计算
@override
Widget build(BuildContext context) {
  final result = heavyComputation();
  return Text(result);
}

// 好：使用 compute 或缓存
@override
Widget build(BuildContext context) {
  return FutureBuilder(
    future: compute(heavyComputation, null),
    builder: (context, snapshot) => Text(snapshot.data ?? ''),
  );
}

图片优化

  • 使用 cached_network_image 缓存网络图片
  • 使用合适分辨率的图片
  • 避免在列表中使用大图片

性能分析工具

  • Dart DevTools：性能分析、内存分析
  • Flutter Inspector：查看 Widget 树
  • Performance Overlay：显示帧率和 GPU 使用
// 开启 Performance Overlay
MaterialApp(
  showPerformanceOverlay: true,
  // ...
);

总结

性能优化的核心是减少不必要的重建和布局计算。善用 Flutter 提供的分析工具，找到性能瓶颈并针对性优化。
