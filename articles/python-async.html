深入理解 Python 异步编程

Python 2026-02-28 · 12 分钟阅读

从 asyncio 到协程，全面解析 Python 异步编程的核心概念与最佳实践。

为什么需要异步？

Python 的 GIL（全局解释器锁）使得多线程在 CPU 密集型任务中效率不高。但对于 I/O 密集型任务（网络请求、文件读写、数据库查询），异步编程可以显著提升性能。

协程基础

协程是异步编程的核心。使用 async def 定义协程函数：

import asyncio

async def fetch_data():
    print("开始获取数据...")
    await asyncio.sleep(1)  # 模拟 I/O 操作
    print("数据获取完成!")
    return {"status": "ok"}

# 运行协程
result = asyncio.run(fetch_data())

asyncio 核心概念

事件循环（Event Loop）：异步任务的调度中心，负责管理和执行协程。

Task：协程的封装，可以在事件循环中调度执行。

async def main():
    # 并发执行多个任务
    task1 = asyncio.create_task(fetch_data())
    task2 = asyncio.create_task(fetch_data())
    
    result1, result2 = await asyncio.gather(task1, task2)
    print(result1, result2)

asyncio.run(main())

async/await 最佳实践

  • 使用 asyncio.gather() 并发执行多个任务
  • 使用 asyncio.wait() 获取更精细的控制
  • 避免在协程中使用 time.sleep()，使用 asyncio.sleep()
  • 使用 asyncio.run() 作为入口点

异步上下文管理器

class AsyncContextManager:
    async def __aenter__(self):
        print("进入上下文")
        return self
    
    async def __aexit__(self, exc_type, exc_val, exc_tb):
        print("退出上下文")

async def main():
    async with AsyncContextManager() as cm:
        print("执行操作")

总结

Python 异步编程通过协程和事件循环实现了高效的并发 I/O 处理。掌握 async/await 语法和 asyncio 模块，可以显著提升程序性能。
