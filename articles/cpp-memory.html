C++ 内存管理深度解析

C/C++ 2026-02-15 · 18 分钟阅读

从栈与堆到智能指针，深入理解 C++ 内存管理的核心概念与现代实践。

栈与堆

栈（Stack）：自动管理，函数结束时自动释放。适合小型、生命周期明确的对象。

堆（Heap）：手动管理，需要显式分配和释放。适合大型、生命周期不确定的对象。

void example() {
    int a = 10;           // 栈上分配
    int* p = new int(20); // 堆上分配
    delete p;             // 手动释放
}

RAII 原则

RAII（Resource Acquisition Is Initialization）是 C++ 内存管理的核心原则：资源的获取与对象的生命周期绑定。

class Resource {
public:
    Resource() { /* 获取资源 */ }
    ~Resource() { /* 释放资源 */ }
};

void useResource() {
    Resource r;  // 构造时获取资源
    // 使用 r...
}  // 析构时自动释放资源

智能指针

现代 C++ 推荐使用智能指针自动管理内存：

std::unique_ptr

独占所有权，不可拷贝：

#include <memory>

std::unique_ptr<int> p1 = std::make_unique<int>(10);
std::unique_ptr<int> p2 = std::move(p1);  // 转移所有权

std::shared_ptr

共享所有权，引用计数：

auto p1 = std::make_shared<int>(10);
auto p2 = p1;  // 引用计数 +1
// 当所有 shared_ptr 销毁时，内存自动释放

std::weak_ptr

弱引用，不增加引用计数，用于打破循环引用：

std::weak_ptr<int> wp = p1;
if (auto sp = wp.lock()) {
    // 成功获取 shared_ptr
}

常见内存问题

  • 内存泄漏：分配后未释放
  • 悬空指针：访问已释放的内存
  • 双重释放：同一内存释放两次
  • 缓冲区溢出：访问越界

最佳实践

  • 优先使用栈内存
  • 使用智能指针替代裸指针
  • 使用 std::make_unique 和 std::make_shared
  • 避免循环引用，使用 std::weak_ptr
  • 使用工具检测内存泄漏（Valgrind、AddressSanitizer）

总结

掌握 C++ 内存管理是成为优秀 C++ 开发者的必经之路。现代 C++ 的智能指针大大简化了内存管理，但理解底层机制仍然重要。
