Python 描述符协议：property 背后的魔法

Python 2026-03-09 · 20 分钟阅读

@property、@staticmethod、@classmethod 都基于同一个底层机制。理解描述符协议，你就理解了 Python 属性访问的本质。

描述符是什么

描述符（descriptor）是实现了 __get__、__set__ 或 __delete__ 方法的对象。当一个描述符被作为类属性存储时，它的这些特殊方法会在属性访问时被调用。

最简单的例子：

class Descriptor:
    def __get__(self, obj, objtype=None):
        print(f"__get__ called: obj={obj}, objtype={objtype}")
        return "descriptor value"

    def __set__(self, obj, value):
        print(f"__set__ called: obj={obj}, value={value}")

class MyClass:
    attr = Descriptor()

obj = MyClass()
print(obj.attr)      # 触发 __get__
obj.attr = 42        # 触发 __set__

注意：描述符必须是类属性，如果是实例属性，它不会生效。

Data Descriptor vs Non-Data Descriptor

这是理解描述符优先级的关键：

  • Data Descriptor：同时定义了 __get__ 和 __set__（或 __delete__）。
  • Non-Data Descriptor：只定义了 __get__。

属性查找的优先级（从高到低）：

  1. Data Descriptor
  2. 实例的 __dict__ 中的属性
  3. Non-Data Descriptor
  4. 类的 __dict__ 中的普通属性

这就是为什么方法（method）可以被实例属性覆盖：

class C:
    def method(self):
        return "from class"

c = C()
print(c.method())    # "from class"

# method 是 non-data descriptor（只有 __get__）
# 所以实例的 __dict__ 优先级更高
c.method = lambda self: "from instance"
print(c.method())    # "from instance"

# 但 data descriptor 不能被覆盖
class D:
    @property
    def value(self):
        return "from descriptor"

d = D()
d.__dict__['value'] = "from instance"
print(d.value)       # 仍然是 "from descriptor"，因为 property 是 data descriptor

手写 property

property 其实就是用描述符实现的。简化版实现：

class MyProperty:
    def __init__(self, fget=None, fset=None, fdel=None):
        self.fget = fget
        self.fset = fset
        self.fdel = fdel

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        if self.fget is None:
            raise AttributeError("unreadable attribute")
        return self.fget(obj)

    def __set__(self, obj, value):
        if self.fset is None:
            raise AttributeError("can't set attribute")
        self.fset(obj, value)

    def __delete__(self, obj):
        if self.fdel is None:
            raise AttributeError("can't delete attribute")
        self.fdel(obj)

    def getter(self, fget):
        return type(self)(fget, self.fset, self.fdel)

    def setter(self, fset):
        return type(self)(self.fget, fset, self.fdel)

# 使用
class Person:
    def __init__(self, name):
        self._name = name

    @MyProperty
    def name(self):
        return self._name

    @name.setter
    def name(self, value):
        if not value:
            raise ValueError("name cannot be empty")
        self._name = value

p = Person("Alice")
print(p.name)    # Alice
p.name = "Bob"   # 正常设置

关键点：property 同时实现了 __get__ 和 __set__，所以它是 data descriptor，优先级高于实例的 __dict__。

staticmethod 和 classmethod 的秘密

这两个装饰器也是描述符，但它们是 non-data descriptor：

class MyStaticMethod:
    def __init__(self, func):
        self.func = func

    def __get__(self, obj, objtype=None):
        # 返回原函数，不绑定任何对象
        return self.func

class MyClassMethod:
    def __init__(self, func):
        self.func = func

    def __get__(self, obj, objtype=None):
        # 绑定到类
        if objtype is None:
            objtype = type(obj)
        import functools
        return functools.partial(self.func, objtype)

class C:
    @MyStaticMethod
    def static_func():
        return "static"

    @MyClassMethod
    def class_func(cls):
        return f"class: {cls.__name__}"

print(C.static_func())      # static
print(C().static_func())    # static（不自动传 self）

print(C.class_func())       # class: C
print(C().class_func())     # class: C（自动传 cls）

实际应用：懒加载属性

描述符的一个实用场景是懒加载（lazy loading）：

class LazyProperty:
    def __init__(self, func):
        self.func = func
        self.name = func.__name__

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        # 只执行一次，然后缓存到实例 __dict__
        value = self.func(obj)
        obj.__dict__[self.name] = value
        return value

# 使用
class DataProcessor:
    def __init__(self, data):
        self.raw_data = data

    @LazyProperty
    def processed(self):
        print("Processing... (this should only happen once)")
        return [x * 2 for x in self.raw_data]

dp = DataProcessor([1, 2, 3])
print(dp.processed)   # Processing... 然后 [2, 4, 6]
print(dp.processed)   # 直接 [2, 4, 6]，没有 "Processing..."

# 原理：第一次访问时，__get__ 被调用，计算结果并存入 __dict__
# 之后访问，由于 lazy_property 是 non-data descriptor，
# 实例的 __dict__ 优先级更高，直接返回缓存值

实际应用：类型检查

class Typed:
    def __init__(self, name, expected_type):
        self.name = name
        self.expected_type = expected_type

    def __set__(self, obj, value):
        if not isinstance(value, self.expected_type):
            raise TypeError(f"Expected {self.expected_type}, got {type(value)}")
        obj.__dict__[self.name] = value

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return obj.__dict__.get(self.name)

def typed(cls):
    # 自动为所有 annotation 添加类型检查
    hints = cls.__annotations__
    for name, expected_type in hints.items():
        setattr(cls, name, Typed(name, expected_type))
    return cls

@typed
class Person:
    name: str
    age: int

p = Person()
p.name = "Alice"      # OK
p.age = 30            # OK
p.age = "thirty"      # TypeError: Expected <class 'int'>, got <class 'str'>

总结

  • 描述符是实现了 __get__、__set__ 或 __delete__ 的对象
  • Data descriptor（有 __set__）优先级 > 实例 __dict__ > Non-data descriptor
  • @property、@staticmethod、@classmethod 都是基于描述符
  • 方法本质是函数对象实现了描述符协议，自动绑定 self/cls
  • 描述符可以实现懒加载、类型检查、ORM 字段等高级功能
